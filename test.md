# 빌드 테스트 문서

> 이 파일은 `build.py` 가 마크다운을 책 리더 HTML 로 잘 바꾸는지 확인하려고 만든 테스트예요.

## 한눈에 (BLUF)

이 파일을 `build.py` 에 넣으면, AI 가 HTML 을 건드리지 않고도 책 리더가 생성돼야 해요. `[사실]`

- 입력: 이 마크다운 한 편
- 출력: 단일 HTML 한 파일
- 변환: 스크립트만 실행 `[사실]`

## 표와 근거 라벨

| 요소 | 지원 | 비고 |
|---|---|---|
| 헤딩 | 예 | H1~H3 가 챕터로 |
| 표 | 예 | 지금 이 표 |
| 라벨 칩 | 예 | 아래 줄 |

근거 라벨은 색칩으로 나와야 해요: `[사실]` 검증됨 · `[추정]` 근거 있으나 불확실 · `[의견]` 주관 판단 · `[모름]` 미확인.

## 코드와 인용

```
python build.py test.md test.html
```

> 인용도 제대로 렌더되는지 봅니다.

마지막 문단이에요. **굵게**와 *기울임* 이 살아있는지 확인하고, 이 H2 가 새 페이지(챕터)로 잘 끊기는지도 봅니다.
