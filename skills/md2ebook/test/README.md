# 테스트

리더(파서 + 동작)의 회귀 가드. 렌더/동작 규칙을 바꾸면 해당 테스트의 기대값도 같이 갱신한다.

## 실행

```sh
node test/parser.test.js     # 파서(순수 로직) — 의존성 없음
node test/search.spec.js     # 리더 동작(브라우저) — playwright-core + 시스템 Chrome 필요
```

`search.spec.js` 의 크롬 경로는 `PLAYWRIGHT_CHROME` 환경변수로 덮어쓸 수 있다.
정본 dir 에 `node_modules` 가 없으면 `extension/node_modules/playwright-core` 를 자동 재사용한다.

## 커버 케이스

### parser.test.js — `src/app.js` 의 `parseMarkdown` 단언
- **1:1 줄바꿈**: 줄바꿈 1개→`<br>`, 빈 줄→문단 분리, 3줄 문단.
- **목록**: lazy 이어쓰기, lazy 2줄, 빈 줄 연속문단, 중첩+연속문단, 중첩 하위 lazy, ul↔ol 분리, 불릿 뒤 비들여쓰기, 체크박스, 체크박스+lazy.
- **블록**: 헤딩, 인용(다줄→`<br>`·빈줄 분리), 표, 굵게/기울임/코드, 인라인코드 내 파이프.
- **회귀 버그**: 고아 표행 무한루프(→단락 처리), 코드펜스 줄바꿈 verbatim(`<br>` 변환 금지).
- **적대 엣지**: 빈 입력, 빈 줄만.

### search.spec.js — 헤드리스 브라우저 동작
- **검색**: Ctrl+F 토글, 'TARGET' 6건, 첫 결과 `1/6`, prev wrap(`1/6`→`6/6`), next 진행 `2..6/6`, 결과가 여러 페이지 분포·점프 페이지 유효, 오검색 `없음`·마크 0, 닫으면 하이라이트 정리.
- **기존 기능 회귀**: 화살표 페이지 이동(다음/이전), Esc 가 TOC 시트 닫기, 설정 변경(글자 크기) reflow 정상, 스크롤 모드 전체 렌더, **JS 런타임 에러 0**.

> 페이지 분할 결과(페이지 번호)는 폰트·뷰포트에 따라 달라지므로 특정 번호로 고정하지 않는다 — 카운터·분포·유효성으로 검증한다.
